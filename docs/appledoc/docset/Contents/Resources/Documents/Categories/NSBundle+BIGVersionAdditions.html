NSBundle(BIGVersionAdditions) Category Reference

Declared in NSBundle+BIGVersionAdditions.h

Overview

Helper categories to report the app’s bundle version.

This category assumes a four part canonical version number with the first three parts being read from the info.plist CFBundleShortVersionString key (marketing version, e.g. 1.2.3) and the fourth part being read from the info.plist CFBundleVersion key (technical / build version e.g. 4567). The versions combined make up an app store compliant version e.g. 1.2.3.4567. All versions must be incrementally higher reading from left to right in order to be accepted by the app store. e.g. 1.2.3.4567 can’t be submitted after 1.2.3.5000, don’t screw it up.
